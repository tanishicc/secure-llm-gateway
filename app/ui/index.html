Secure LLM Gateway

Interactive AI Security Demo (Prompt Injection • DLP • Policy • Audit)

What this is

This project is an AI security gateway that sits between users and an LLM. It detects prompt injection, blocks risky prompts in strict mode, and redacts sensitive data using DLP (Data Loss Prevention) — all while producing audit logs you can review like a real enterprise control layer.
Control 1

Prompt Injection Protection

Detects “ignore instructions / reveal system prompt / jailbreak” patterns and blocks them in strict mode.
Control 2

DLP Redaction (Input + Output)

Redacts secrets like API keys/emails so they don’t leak into logs or responses.
Control 3

Auditability + Self-Tests

Selftest proves controls (4/4). Audit logs are JSONL for SIEM-style ingestion.
Tip: You’re currently running in Mock Mode (offline). The gateway still enforces the same controls — it just uses a simulated model response.

How it works (quick):
1) Prompt comes in → 2) Risk scoring + DLP → 3) Policy ALLOW/BLOCK → 4) Model response → 5) Output DLP → 6) Audit log written.

Tutorial: What you’re seeing

          

            This tutorial explains how the gateway processes a prompt. You can follow along while using the Live Demo.
          


          

            

              
Step 1

              
Auth + Rate limit

              

                Requests must include x-api-key. The gateway also rate-limits to prevent abuse.
              

            

            

              
Step 2

              
DLP on input

              

                If the prompt contains emails/keys, it can be redacted before being sent onward.
              

            

            

              
Step 3

              
Risk scoring

              

                The scorer flags patterns like “ignore previous instructions” and assigns a risk score + reasons.
              

            

            

              
Step 4

              
Policy enforcement

              

                In strict mode, prompt injection is blocked immediately.
              

            

            

              
Step 5

              
Model response

              

                In mock mode, the “model” is simulated so you can demo without paying for APIs.
              

            

            

              
Step 6

              
DLP on output

              

                If the output contains secrets, it is redacted before returning to the user.

About this project

          

            Secure LLM Gateway is a security middleware layer designed for enterprise LLM adoption.
            It shows how you can wrap LLM calls with enforceable controls: authentication, rate limiting,
            prompt risk scoring, policy enforcement, DLP redaction, and audit logging.
          


          

            
Why I built this

            

              I built this to demonstrate practical, production-style AI security patterns.
              Enterprises need controls like prompt-injection blocking and DLP to safely deploy LLM features.
              This project is a compact, testable example of that control plane.
